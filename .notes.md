This is my project for learning how to use Claude to code as the primary author of a codebase. This was inspired after watching these YouTube videos from prominent software philosophers who all expressed enthusiasm for these new tools and the opportunities they present.

- [Martin Fowler](https://www.youtube.com/watch?v=CQmI4XKTa0U)
- [DHH](https://www.youtube.com/watch?v=JiWgKRgdgpI)
- [Kent Beck](https://www.youtube.com/watch?v=aSXaxOdVtAQ)

This file captures my notes for myself and future conversations about what I learned.

Firstly, Claude and agentic coding have evolved a ton in the last year. I read through a good bit of the Claude Code documentation and several new concepts jumped out at me since the last time I looked into this domain seriously.

1. The workflow and concepts are much more solidified than before, there's clear guidance about concepts and skills to develop to take advantage of Claude (and agentic coding tools generally). Explanations for first principles and primary concerns as a user of the tool are also laid out plainly.
2. Keeping track of the context window's capacity is a first principle skill to avoid exhausting tokens.
3. Common problems are clearly enumerated along with recommended solutions (using Claude.md, Agents, and Skills)
4. The CLI seems like the primary interface now, which I suppose makes sense because editing the code itself is less emphasized in this workflow. I'm now pair programming with a developer (Claude) that's writing the code, rather than writing the code myself and expecting it to help me.
5. A TDD mindset is helpful here, as a key recommendation is giving Claude tools to be able to measure its own success as it's iterating. Telling it how to tell when it's been successful seems critical.
